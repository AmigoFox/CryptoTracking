Цель проекта «Трекер криптовалют»

Автоматизация данных:
Сбор курсов криптовалют через CoinMarketCap API и сохранение в PostgreSQL.
Преобразовать JSON-ответы в типизированные модели (например, Pydantic) и проверить корректность полей (price, market_cap, volume).
Организовать очередь задач (Celery или APScheduler) для надёжного выполнения запросов каждые N минут.
Сохранить результаты в PostgreSQL: таблицы cryptocurrencies (метаданные) и price_history (timestamp, crypto_id, price_usd, volume_24h), с индексами по crypto_id и timestamp.

Визуализация:
Построение графиков через QuickChart для анализа динамики курсов.
Формировать HTTP-запросы к QuickChart API с параметрами: тип графика, выбранный период (hourly, daily, weekly) и набор ценовых точек.
Автоматически генерировать URL или base64-код изображения графика и кешировать результаты (Redis или HTTP-кеш) для снижения задержек при повторном запросе.

Доступность:
Удобный интерфейс для выбора криптовалюты и просмотра графиков.
Реализовать REST-API (FastAPI) с эндпоинтами:
/cryptocurrencies — список доступных монет,
/charts/{crypto_id} — параметры period, type и ответ в виде ссылки на график или встраиваемого изображения.
Предусмотреть валидацию входных параметров и отдачу понятных HTTP-кодиров ответов (200, 400, 500).
